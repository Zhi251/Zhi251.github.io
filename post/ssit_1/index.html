What is SSIT(Server-Side Template Injection)?

SSTI（Server-Side Template Injection，伺服器端模板注入）是一種網頁應用程式的安全漏洞。有些網頁框架常使用模板引擎（如 Jinja2、Twig、Freemarker 等）來動態產生 HTML。模板引擎允許在 HTML 中嵌入變數與邏輯。利用SSTI 漏洞可以進行以下攻擊

  1. 讀取伺服器上的敏感檔案(如設定檔、密碼)
  2. 遠端程式碼執行（RCE）
  3. 獲取環境變數存取資料庫等

解題

檢測

打開題目網頁 對其進行SSIT漏洞測試,發現該網站使用的是jinja2(python)的腳本引擎

對其進行模板注入

獲取該使用者帳號id

{{request.application.__globals__.__builtins__.__import__('os').popen('id').read()}}

  1. request — Jinja2 模板中預設可存取的 Flask request 物件，代表當前的 HTTP 請求。
  2. .application — 從 request 取得 Flask 應用程式實例（即 Flask app 物件）。
  3. .globals — 存取該應用程式物件所屬函式的全域變數字典。這是 Python 函式物件的內建屬性，包含了該模組作用域中所有的全域名稱。
  4. .builtins — 從全域變數字典中取得 Python 的內建函式集合，包含 print、open、import 等核心函式。
  5. .import(‘os’) — 呼叫內建的 import 函式來動態載入 os 模組。這是關鍵的一步，因為 os 模組提供了與作業系統互動的能力。
  6. .popen(‘id’) — 呼叫 os.popen()，在伺服器上開啟一個子行程執行系統指令 id（Linux/Unix 指令，回傳當前使用者的 UID、GID 等資訊）。
  7. .read() — 讀取指令的輸出結果，讓它顯示在網頁回應中。
